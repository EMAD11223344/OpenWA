# OpenWA Baileys HF Fix — Implementation Plan

## Status: APPROVED + IN PROGRESS

## Root Cause

The live HF log shows the chain:
1. ✅ `is unauthenticated — resetting auth dir to generate fresh QR`
2. ✅ `Using latest Baileys WA version: 2.3000.1035194821`
3. ✅ `Baileys socket created for session ...`
4. ❌ **60s later → `SSL alert 0` → `code=408` → QR event never fires**

WhatsApp rejects the raw `wss://` TLS handshake from HF datacenter IPs.
The QR event only fires AFTER WhatsApp responds with the challenge — but it never does, so Baileys emits `0` QR codes.

## Cause Tracing

| Comparison | WhatsApp Web.js | Baileys (current HEAD `9ec0819`) |
|---|---|---|
| TLS stack | Chromium browser | Raw `wss://` + Node default TLS |
| Proxy | `--proxy-server` Puppeteer arg | `getProxyConfig()` returns `{}` |
| HF datacenter IPs | Works (browser fingerprint) | Fail — `SSL alert 0` |

The current `src/engine/adapters/baileys.adapter.ts:275` has:
```typescript
private getProxyConfig(): Record<string, unknown> {
  // Proxy support can be added via config
  return {};
}
```

A previous commit (`624e06a` — `fix(baileys): add direct proxy-agent deps + WHATSAPP_PROXY_URL global env + proxy hint in logs`) had working proxy code, but was wiped in later commits.

Also: the MACOS postinstall patch (`replace('UserAgent.Platform.WEB','UserAgent.Platform.MACOS')`) added between `056092e` and `9ec0819` exists now but did NOT exist in the proven working commit `056092e`.

## Required Code Changes

### 1. `package.json` — Add deps + simplify postinstall

```json
"dependencies": {
  ...,
  "@whiskeysockets/baileys": "7.0.0-rc13",
  "https-proxy-agent": "^7.0.6",
  "socks-proxy-agent": "^8.0.5",
  ...,
},
"postinstall": "[ -d dashboard ] && npm run dashboard:install || true"
```

Remove the `node -e "..." WEB→MACOS replace` from postinstall.

### 2. `src/engine/adapters/baileys.adapter.ts` — Add proxy fields + real `getProxyConfig()`

- Add `proxyUrl`/`proxyType` to `BaileysAdapterConfig`
- Wire them as class fields; default `proxyType = 'http'`
- Replace empty `getProxyConfig()` with SOCKS/HTTPS agent builder:
  - `socks4` / `socks5` → `SocksProxyAgent(this.proxyUrl)`
  - `http` / `https` → `HttpsProxyAgent(this.proxyUrl)`
  - Forward as `{ agent }` to `makeWASocket`
  - On error → log and return `{}`
- Add log fields: `proxyEnabled`, `proxyType` on socket-creation log
- Add warn log if proxy is missing on HF datacenter blocks

### 3. `src/engine/engine.factory.ts` — Forward proxy to fallback Baileys creator

Two paths create `BaileysAdapter`:
- Plugin path (line 100-103) — already forwards proxy to `createEngine` ✓
- Fallback path (line 109-124) — does NOT forward proxy
- `createFallbackEngine` (line 145-152) — does NOT forward proxy

Add `proxyUrl`/`proxyType` to both fallback Baileys construction calls.

### 4. `src/plugins/engines/baileys/index.ts` — Pass proxy to BaileysAdapter

Update `createEngine(config)` to read `config.proxyUrl` and `config.proxyType` and pass them when constructing `BaileysAdapter`.

### 5. `src/modules/session/session.service.ts` — Resolve proxy env var

In `initializeEngine()` (line 262):
```typescript
const envProxyUrl = process.env.WHATSAPP_PROXY_URL || process.env.HF_PROXY_URL || null;
const envProxyType = (process.env.WHATSAPP_PROXY_TYPE || 'http') as 'http'|'https'|'socks4'|'socks5';
const resolvedProxyUrl = session.proxyUrl || envProxyUrl || undefined;
const resolvedProxyType = (session.proxyType as typeof envProxyType) ||
  (envProxyUrl ? envProxyType : undefined);
```

Pass `resolvedProxyUrl` / `resolvedProxyType` to `engineFactory.create()`.

### 6. `.env.example` and `.env.minimal` — Comment hint

```bash
# WhatsApp proxy (REQUIRED on HF datacenter IPs)
# WHATSAPP_PROXY_URL=socks5://user:pass@residential-proxy.example.com:1080
# WHATSAPP_PROXY_TYPE=socks5
```
